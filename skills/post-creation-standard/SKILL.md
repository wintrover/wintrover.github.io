## When to Use

- 새로운 블로그 포스트(Project 또는 Company)를 작성할 때
- 기존 포스트의 구조를 개선하거나 수정할 때

## Rules

- **이중 언어 정책**: **모든 포스트는 영어(English)와 한국어(Korean) 두 언어로 작성해야 한다.**
  - 영어 포스트: `src/posts/project/` 또는 `src/posts/company/`에 저장
  - 한국어 포스트: `src/posts/ko/project/` 또는 `src/posts/ko/company/`에 저장
  - 동일한 파일명(`YYYY-MM-DD-N.md`) 사용
  - 내용은 동일하게 유지하되 자연스러운 번역 제공
- **프로젝트 명칭 정책**: **포스팅 내용에서 프로젝트명은 일관되게 지칭한다.**
  - `Axiom Enterprise` 폴더명을 그대로 사용하지 말고, `Axiom`으로만 지칭
  - 예: "Axiom Enterprise was built" → "Axiom was built"
- **GitHub Actions 확인**: `gh` 명령어만 사용한다. (예: `gh run list`, `gh run view`)
- **Git 명령어 페이저 비활성화**: 모든 git 명령어는 반드시 `git --no-pager <subcommand>` 형식으로 실행한다.
- **공식 문서 탐색**: Context7은 MCP가 아닌 CLI(`ctx7` 또는 `npx ctx7`)만 사용한다.
- **파일명**: `YYYY-MM-DD-N.md` 형식을 준수한다. (예: `2026-02-01-18.md`)
- **Frontmatter**: 아래 필드를 반드시 포함한다.
  - `layout: post`
  - `title`: 명확하고 호기심을 유발하는 제목
  - `date`: `YYYY-MM-DD HH:MM:SS -0000`
  - `tags`: 3개 이상의 관련 태그
  - `category`: `Project` 또는 `Company Work`
  - `description`: 1-2문장의 핵심 요약
- **본문 구성**: 포스팅 형식과 섹션 구성은 자유롭게 작성한다.
- **분량**: 전달력을 해치지 않는 범위에서 자유롭게 작성한다.
- **부분 수정 불변성(기본값)**: 부분 수정 시 동일 구간의 토씨·문장부호·줄바꿈을 변경하지 않는다.
- **역슬래시 보존**: 줄 끝 `\`는 가독성 제어 기호로 간주하며 삭제·이동·정규화하지 않는다.
- **구조 마커 보존(기본값)**: `<hr ... />`, `##`, `###`, 섹션 번호 표기(`1)`, `1.`)를 기존 형식 그대로 유지한다.
- **표현 보존**: 사용자가 명시한 핵심 표현(예: 괄호 구문, 도구명 나열 문장)은 축약·일반화하지 않는다.
- **흐름 블록 레이아웃 보존**: `요구사항 입력`, `→` 단계 연결 같은 흐름형 문장은 빈 줄 간격과 단일행/다중행 배치를 원문 그대로 유지한다.
- **절차 단락 번호화 보존**: 정형 검증/프로세스 설명이 번호 목록(`1.`~`N.`)으로 정리된 경우 동일한 번호 목록 형식을 유지한다.
- **예시 문장 종결부 보존**: `예를 들어:\` 다음의 조건절/결과절에서 쉼표, 마침표, 줄 끝 `\`를 임의로 통일하거나 제거하지 않는다.

## Checklist

- [ ] Frontmatter가 모든 필수 필드를 포함하고 있는가?
- [ ] 카테고리가 정확히 `Project` 또는 `Company Work`로 지정되었는가?
- [ ] **영어 포스트가 `src/posts/project/` 또는 `src/posts/company/`에 위치하는가?**
- [ ] **한국어 포스트가 `src/posts/ko/project/` 또는 `src/posts/ko/company/`에 위치하는가?**
- [ ] **두 언어 버전 모두 작성되었는가?**
- [ ] **두 버전의 파일명이 동일한가?**
- [ ] **프로젝트 명칭이 일관된 형태로 지칭되었는가?** (예: `Axiom Enterprise` → `Axiom`)
- [ ] 사용자가 지정한 비수정 구간의 조사/기호/줄바꿈/`\`가 보존되었는가?
- [ ] 사용자가 지정한 구분선과 제목 계층 표기가 유지되었는가?
- [ ] 흐름 블록(`요구사항 입력`, `→`)의 빈 줄/배치/`\`가 원문과 일치하는가?
- [ ] 절차 번호 목록과 예시 문장 종결부(쉼표·마침표·`\`)가 유지되었는가?
