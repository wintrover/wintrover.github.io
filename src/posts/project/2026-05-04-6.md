## 1. Goal

We were removing the last remaining bash wrapper (prek_hook_wrapper.sh) and migrating all of Axiom's quality gates to pure Nim.

When this stage ended:

- Every gate is a Nim binary
- Git hook → Nim runner direct
- Zero shell dependency

## 2. Problem Emerged

The migration itself wasn't hard.

The problem came after.

```
gcc: fatal error: cannot execute 'cc1'
```

Strange.

- gcc was installed
- Previously working code
- Only "gate execution method" changed

Meaning:

> Code stayed the same, but execution broke

## 3. Wrong Hypothesis

First thought:

- gcc path issue?
- PATH corrupted?
- Some env variable missing?

Following this direction usually leads to:

> Hardcode path → Temporary fix → Breaks again later

## 4. Real Cause

The problem was more fundamental.

```nim
execCmdEx(cmd, env = customEnv)
```

This one line.

When passing env in Nim:

- It doesn't "add" to the existing environment
- **It replaces everything**

Result:

- PATH gone
- HOME gone
- All toolchain access blocked

Meaning:

> We didn't modify the environment.
> We isolated the entire process.

## 5. Solution

The fix itself was simple.

```nim
for (k, v) in envPairs():
  envTable[k] = v

for (k, v) in envVars:
  envTable[k] = v
```

Restore parent environment first,
then overlay only what the gate needs.

## 6. Important Choice

A fork existed here.

**Choice 1**
- Restore PATH only
- Add only needed variables

**Choice 2**
- "Fully" inherit the environment
- Apply only overrides

We chose the second.

Reason is simple.

> The problem wasn't PATH.
> It was the execution context itself.

## 7. Result

After this change:

- All gates run normally
- gcc / cc1 issue gone
- Environment errors unreproducible

And more important changes:

- Shell wrapper completely removed
- Gate runner unified (Nim)
- Execution path 100% internalized

## 8. Reflection

Initial goal was simple.

> "Remove bash"

But what we actually did was different.

**Revoked execution rights**

Before:
- Some execution delegated to shell
- Environment implicit

After:
- All execution inside Nim
- Environment explicit

## 9. Summary

One-line summary:

```
Shell-based execution → Deterministic runtime execution
```
