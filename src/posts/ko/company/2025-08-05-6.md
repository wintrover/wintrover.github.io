## 🎯 프로젝트 개요

회사 플랫폼에서 사용할 **얼굴 인식 기반 사용자 인증 시스템**을 개발하게 되었습니다. 중요한 프로젝트를 사실상 단독으로 리딩하게 되었고, 단순한 기능 추가가 아니라 **실시간 처리**, **다양한 환경(조명/각도/안경 등) 대응**, **웹 기반 구현**까지 여러 기술적 과제를 포함했습니다.

## 🚀 Phase 1: 핵심 기술 개발 & 성능 최적화 (7/30)

### Deepface + MediaPipe 하이브리드 아키텍처 구현

처음에는 Deepface의 ArcFace 모델을 기반으로 실시간 얼굴 인식 프로토타입을 만들었습니다. 하지만 초기 테스트에서 다양한 환경(조명, 각도, 안경 등)에서 인식률이 크게 떨어지는 문제가 드러났습니다.

**해결**: MediaPipe의 3D 랜드마크 기술을 결합해, 다양한 조건에서의 인식률을 크게 개선했습니다.

![Facial Recognition System Architecture](../assets/images/06/facial-recognition-architecture.svg)
*Deepface와 MediaPipe를 결합해 정확도를 끌어올린 하이브리드 아키텍처*

### 성능 최적화 작업

- **처리 속도**: 5.5 FPS까지 개선
- **메모리 관리**: 기술 통합 과정에서 발생한 메모리 누수 해결
- **실시간 처리**: 웹캠 스트리밍 기반 실시간 얼굴 탐지 구현

## 🏗️ Phase 2: KYC 시스템 아키텍처 설계 & 요구사항 분석 (7/31 ~ 8/1)

### 요구사항 수집 & 시스템 설계

개발 과정 공유/리포팅을 통해 KYC 절차에서 필요한 요구사항을 구체화했습니다.

- **사용자 가이드**: “왼쪽을 보세요/오른쪽을 보세요” 같은 실시간 안내
- **데이터 저장**: 다중 프레임 캡처 및 비디오 저장
- **정밀 분석**: 전체 얼굴뿐 아니라 눈/코/입 등 특정 부위 단위의 유사도 분석

### 아키텍처 전환

데스크톱 환경이 아니라, 브라우저 기반 웹캠 시스템으로 구현 방향을 전환했습니다.

### 기술 검토 & 협업 환경 준비

- **특허 분석**: ETRI/애플 특허를 비교해 기술적 유효성을 검토
- **협업 환경**: Docker + GitHub 기반으로 개발 환경 공유 체계 구축
- **버전 관리**: 협업 효율을 위한 브랜칭 전략 수립

## 🌐 Phase 3: 웹 기반 구현 & 기술적 난관 해결 (8/4 ~ 8/5)

### 백엔드 기술 스택 전환

웹 환경에서 웹캠 기능을 안정적으로 처리하기 위해 백엔드를 Python에서 Express.js로 마이그레이션했습니다.

### MediaPipe 웹 연동 이슈 해결

MediaPipe를 웹으로 붙이는 과정에서 여러 문제가 발생했습니다.

#### 3.1 CSP(Content Security Policy) 충돌
```javascript
// Problem: CDN resources blocked by CSP
// Solution: Updated helmet middleware configuration

  contentSecurityPolicy: {
    directives: {
      scriptSrc: ["'self'", "https://cdn.jsdelivr.net"],
      connectSrc: ["'self'", "https://cdn.jsdelivr.net"],
      workerSrc: ["'self'", "blob:"]
    }
  }
}));
```

#### 3.2 파일 로딩 타이밍 이슈
- WASM 파일 로딩 지연 문제 해결
- 비동기 로딩을 위한 Promise 기반 초기화 적용
- 모듈 의존성 관리 최적화

#### 3.3 랜드마크 데이터 처리 오류
```javascript
// Problem: Landmark coordinates returned as normalized values
// Solution: Added logic for pixel coordinate conversion
drawLandmarkPoints() {
  this.ctx.arc(
    landmark.x * this.canvas.width,
    landmark.y * this.canvas.height,
    this.landmarkStyle.radius,
    0,
    2 * Math.PI
  );

```

## 🔧 Phase 4: 디버깅 & 성능 최적화

### 주요 문제와 해결 과정

#### 4.1 Yaw 각도 측정 오류
**문제**: 정면을 보고 있는데 Yaw 각도가 비정상적으로 크게(85도) 측정됨

**원인**: `generateDummyLandmarks` 함수가 만든 기본값(0,0) 랜드마크가 얼굴 너비 계산에 포함됨

**해결**: `calculateAngles`에서 더미 랜드마크를 제외하도록 수정
```javascript
// Exclude dummy landmarks during face width calculation
const validLandmarks = landmarks.filter(landmark =>
  landmark.x !== 0 || landmark.y !== 0
);
const faceWidth = Math.max(...validLandmarks.map(l => l.x)) -
                  Math.min(...validLandmarks.map(l => l.x));
```

#### 4.2 랜드마크 렌더링 성능 이슈
**문제**: 478개 랜드마크 포인트를 실시간으로 렌더링하면서 성능이 저하됨

**해결**:
- 불필요한 console 로그 95% 제거
- Canvas 렌더링 최적화
- requestAnimationFrame 기반 부드러운 애니메이션 적용

#### 4.3 서버 연결 이슈
**문제**: `FaceRecognitionClient`의 `testServerConnection`이 무한 루프를 유발

**해결**: 재시도 제한(20회, 약 2초)과 초기화 실패 시 적절한 에러 처리 추가
