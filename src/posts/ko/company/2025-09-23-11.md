## 왜 마이그레이션했나?

DeepFace로 초기 실험을 진행하면서, GPU 병렬성 및 다중 이미지 처리량에서 현실적인 한계에 부딪혔습니다. InsightFace로 옮기면서 GPU에서 다중 이미지 처리를 표준화할 수 있었고, 이미지당 지연 시간을 줄이며 처리량을 끌어올려 프로덕션 트래픽을 감당할 수 있는 파이프라인에 가까워졌습니다.

## 품질과 운영
- SonarQube로 버그/코드 스멜/핫스팟을 자동 탐지
- SOLID 기반 리팩토링으로 service/repository/gateway 레이어 의존성을 안정화하고 책임을 분리
- Docker 기반 GPU 런타임 구성으로 dev/test/CI 환경을 일관되게 유지
- 변경마다 lint와 라이트한 체크를 수행하는 CI 파이프라인 구축

## 벤치마킹과 시각화
- 객관적 검증을 위해 LFW를 사용(Accuracy, AUC)
- W&B를 도입해 실험을 트래킹하고, 실행(run)별로 지표(정확도 곡선, ROC/AUC)를 시각화해 회귀를 눈에 보이게 만들고 재현성을 확보

## 서비스 통합(React + FastAPI)
- 실제 서비스 연동 과정에서 업로드 버그와 타입 불일치를 수정
- 안정적인 데이터 교환을 위해 명확한 계약(contracts)과 방어적 에러 경로(타임아웃, 잘못된 payload 등)를 정립

## 예시 코드

### 1) InsightFace: GPU 기반 임베딩(단일 이미지)
```python
from insightface.app import FaceAnalysis
import cv2

app = FaceAnalysis(name='buffalo_l')
app.prepare(ctx_id=0, det_size=(640, 640))

img = cv2.imread('face.jpg')
faces = app.get(img)
embeddings = [f.normed_embedding for f in faces]
print(len(embeddings), 'embeddings computed')
```

### 2) CI 파이프라인: lint + build(GitHub Actions)
```yaml
name: ci

```

### 3) W&B로 LFW 지표 로깅
```python
import wandb
from sklearn.metrics import accuracy_score, roc_auc_score

wandb.init(project='face-benchmark', config={'dataset': 'LFW'})
# y_true, y_score prepared from verification pairs
acc = accuracy_score(y_true, (y_score > 0.5).astype(int))
auc = roc_auc_score(y_true, y_score)
wandb.log({'lfw/accuracy': acc, 'lfw/auc': auc})
wandb.finish()
```

### 4) FastAPI 엔드포인트 + React 업로드
```python
# fastapi
from fastapi import FastAPI, UploadFile, File
app = FastAPI()

@app.post('/api/images')
async def upload_image(file: UploadFile = File(...)):
    data = await file.read()
    # validate & store
    return {"ok": True, "size": len(data)}
```

```tsx
// react
async function upload(blob: Blob) {
  const fd = new FormData();
  fd.append('file', blob, 'image.jpg');
  const res = await fetch('/api/images', { method: 'POST', body: fd });
  if (!res.ok) throw new Error('Upload failed');
}
```

## 결과: LFW + W&B(InsightFace–ArcFace)

아래 이미지는 InsightFace(ArcFace)를 LFW 검증 페어에서 평가하면서 Weights & Biases에 기록한 실행 대시보드입니다. ROC/AUC와 accuracy/precision/recall 지표를 epoch별로 시각화하고, threshold drift와 기본 GPU 텔레메트리까지 함께 확인할 수 있습니다. 이 곡선들은 합리적인 결정 임계값(threshold)을 선택하는 데 도움이 되었고, 이후 실험에서 회귀(regression)를 빠르게 감지하는 데도 유용했습니다.
