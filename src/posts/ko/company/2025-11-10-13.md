## 🎯 프로젝트 개요

**배경**: 프로덕션급 KYC(Know Your Customer) 검증 시스템을 처음부터 구축
**기간**: 3개월 집중 개발
**팀 규모**: 사실상 단독 개발(백엔드 인프라 지원 포함)
**비즈니스 임팩트**: 컴플라이언스 및 사용자 온보딩에 핵심

회사 핵심 KYC 시스템 개발을 리딩하게 되었습니다. 단순히 “기술적으로 어려운 과제”가 아니라, 회사가 규제 준수를 유지하면서 사용자 온보딩을 확장할 수 있느냐를 좌우하는 **비즈니스 크리티컬 프로젝트**였습니다. 시스템은 하루 수천 건의 검증 시도를 처리해야 했고, 정확도도 99.9% 수준을 요구했습니다.

## 🏗️ 시스템 아키텍처 설계

처음에는 얼굴 인식 라이브러리를 몇 가지 기준으로 비교/분석했습니다.

## 🔥 Phase 1: 얼굴 인식 듀얼 엔진 구현

### **하이브리드 전략**: 왜 엔진 하나로는 부족했나

실제로 맞닥뜨려 보니, 단일 얼굴 인식 엔진만으로 모든 현실 환경을 커버하기 어려웠습니다. InsightFace는 정확도(99.8%)가 매우 뛰어났지만 조명이 나쁘거나 각도가 틀어질 때 실패했고, OpenCV는 매우 안정적이지만 정확도가 조금 낮았습니다.

**해결**: 상황에 따라 자동으로 엔진을 전환하는 듀얼 엔진 시스템

![Mermaid Diagram](/images/2025-11-10-13-7.png)

**핵심 아이디어**
- **스마트 하드웨어 감지**: GPU/CPU 환경에 자동 적응
- **메모리 관리**: 싱글톤 패턴으로 GPU 메모리 누수 방지
- **폴백 로직**: confidence score 기반 자연스러운 전환

**효과**: 두 엔진의 장점을 결합하면서 성공률이 92% → 99.9%로 뛰었습니다.

## 🎥 Phase 2: 비디오-이미지 유사도 비교

**스마트 샘플링 전략**
![Mermaid Diagram](/images/2025-11-10-13-6.png)

## 📸 Phase 3: 신분증 자동 캡처

### **한글 OCR 난관**: 컴퓨터에게 한글을 읽게 하려면

대부분의 OCR은 한글에서 성능이 떨어졌습니다. Tesseract, EasyOCR, 클라우드 서비스를 테스트했고, 한국어 지원이 의외로 괜찮았던 PaddleOCR을 찾았지만 실제 현업 수준으로 쓰려면 추가 튜닝이 필요했습니다.

**자동 품질 평가 파이프라인**
![Mermaid Diagram](/images/2025-11-10-13-5.png)

**4가지 품질 지표**
- **선명도(Sharpness)**: Laplacian variance로 블러 감지
- **조명(Lighting)**: 과다 노출/반사 없이 고르게
- **각도(Angle)**: 원근 왜곡(perspective distortion) 감지
- **완전성(Completeness)**: 네 모서리 모두 가시성 확인

**결과**: 사용자가 수동으로 타이밍 맞춰 찍는 방식(완료율 60%)에서, 자동 캡처 기반으로 완료율이 95%까지 상승했습니다.

## 🐛 주요 디버깅 과정

### 문제 1: GPU에 숨어 있던 유령 — 메모리 누수

**위기**: 프로덕션에서 1주일쯤 잘 돌던 시스템이, 갑자기 4~6시간마다 주기적으로 죽기 시작했습니다. RAM은 안정적이었는데, 원인은 GPU 메모리였습니다.

**단계별 해결**

1. **문제 인지**
   - 증상: 4~6시간마다 크래시
   - 초기 진단: 메모리 누수
   - 도구: `nvidia-smi`, 시스템 모니터링

2. **가설 검증**
   - 가설 1: RAM 누수 → ❌ RAM 사용량 안정적
   - 가설 2: GPU 메모리 누수 → ✅ GPU 메모리가 지속 증가
   - 근거: 얼굴 인식 호출마다 GPU 메모리 50~100MB 증가

3. **원인 분석**
   - 위치: InsightFace 모델 초기화
   - 문제: 추론 후 GPU context가 해제되지 않음
   - 영향: 누적 할당으로 인한 크래시

4. **해결 적용**
   ```python
   # Memory management workflow
   def process_with_memory_cleanup():
       try:
           # Face recognition operation
           result = insightface_app.process(frame)
           return result
       finally:
           # Critical: Explicit GPU cleanup
           if torch.cuda.is_available():
               torch.cuda.empty_cache()
               torch.cuda.synchronize()
   ```

5. **재발 방지**
   - 임계값 기반 메모리 모니터링
   - 서비스 재시작 자동화
   - 정기 리포팅

**배운 점**: GPU 메모리는 명시적으로 정리해야 합니다. Python GC가 GPU 리소스를 자동으로 해제하지 않아 누적 누수가 생기고, 이는 프로덕션을 무너뜨릴 수 있습니다.

### 문제 2: 시간을 여행하는 비디오 프레임

**이상한 버그**: 유사도 계산 결과가 말이 안 되는 경우가 있었고, 프레임 타임스탬프는 연속인데 실제 프레임은 앞뒤가 섞여 있는 듯 보였습니다.

**디버깅 흐름**

1. **이상 징후 탐지**
   - 증상: 기대한 프레임 진행과 유사도 결과 불일치
   - 근거: 프레임 타임스탬프와 계산 결과가 어긋남
   - 영향: 랜덤한 정확도 저하

2. **원인 추적**
   ```python
   # Problem: OpenCV's internal buffering
   cap.set(cv2.CAP_PROP_POS_FRAMES, target_frame)  # Requested frame
   ret, frame = cap.read()  # Got buffered frame instead!
   ```

3. **해결 적용**
   ```python
   # Frame precision control
   cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)  # Minimize buffer
   cap.set(cv2.CAP_PROP_POS_FRAMES, target_frame)
   ret, frame = cap.read()

   # Validation step
   actual_frame = int(cap.get(cv2.CAP_PROP_POS_FRAMES))
   if actual_frame != target_frame:
       # Handle frame mismatch
   ```

### 문제 3: 동시성 대참사

**붕괴 시나리오**: 동시 사용자 10명만 올려도 결과가 뒤섞이기 시작했습니다. 다른 사람의 유사도 점수가 섞여 나오는 심각한 보안/프라이버시 이슈였습니다.

**대응과 해결**

1. **사고 대응(분 단위)**
   - 즉시 시스템 중단
   - 보안팀 알림
   - 포렌식을 위한 로그 보존

2. **원인 분석(시간 단위)**
   ```python
   # Problem: Shared singleton instance
   class FaceRecognitionService:
       _instance = None  # Shared across all requests! ❌

   # Solution: Service pooling
   class FaceRecognitionPool:
       def __init__(self, pool_size=5):
           self.pool = [FaceRecognitionService() for _ in range(pool_size)]
           self.available = Queue()
       def get_service(self):
           return self.available.get()
       def return_service(self, service):
           self.available.put(service)
   ```

3. **보안 검증**
   - 100+ 동시 요청 멀티스레드 테스트
   - 결과 검증: 오염 없음
   - 성능 테스트: 처리량 유지

4. **프로덕션 세이프가드**
   - 모든 얼굴 인식 작업에 대한 상세 로깅
   - request correlation tracking
   - 이상 징후 자동 감지

**배운 점**: 바이오메트릭 시스템에서 스레드 세이프티는 선택이 아니라 필수입니다. 민감한 데이터를 다룰수록, 처음부터 동시성을 전제로 설계해야 합니다.
