## 🎯 문제 해결에서 기능 완성으로: KYC 프로세스 구현

![KYC Process Workflow](/images/07-kyc-workflow-diagram.png)

*각도 감지와 비디오 캡처 단계로 구성된 KYC 전체 워크플로우*

지난 글에서는 Yaw 각도 측정 정확도를 높이기 위해 얼마나 삽질했는지 공유했습니다. 이번에는 그 결과를 기반으로 **KYC(Know Your Customer)** 시스템의 핵심 기능을 실제 프로세스에 녹이는 작업에 집중했습니다.

가장 먼저 지난주 최대 성과였던 **Yaw 각도 측정 로직**을 KYC 흐름에 적용했습니다. 초기 코드에 남아 있던 `Math.random()` 같은 임시 코드를 제거하고, 실제 **MediaPipe** 랜드마크 데이터를 기반으로 계산 로직을 다듬었습니다. 이제 사용자가 고개를 돌리는 방향에 따라 각도가 실시간으로 정확히 측정되고, 그 값을 기반으로 다음 단계로 넘어가는 안정적인 조건 로직을 만들 수 있었습니다.

![Angle Detection Diagram](/images/07-angle-detection-diagram.png)

위 다이어그램은 얼굴 각도 감지 시스템의 동작 원리를 설명합니다. MediaPipe로 얼굴 랜드마크를 추출한 뒤, 3D 좌표계에서 yaw/pitch/roll을 계산해 사용자가 올바른 방향을 바라보는지 실시간으로 판정합니다.

### 설정 값 예시
- **정면(Front)**: yaw ±10° 이내
- **왼쪽(Left)**: yaw −30° ~ −15°
- **오른쪽(Right)**: yaw +15° ~ +30°

조건이 만족되면 각 단계별로 자동으로 비디오를 캡처하고, 다음 단계로 진행하는 워크플로우를 완성했습니다.

```javascript
// Yaw angle calculation logic in face-detection.js
calculateAngles(landmarks) {
    if (!landmarks || landmarks.length === 0) {
        return { yaw: 0, pitch: 0, roll: 0 };
    }

    // Actual angle calculation using MediaPipe landmarks
    const noseTip = landmarks[1];  // Nose tip
    const leftEye = landmarks[33]; // Left eye
    const rightEye = landmarks[263]; // Right eye

    // Yaw angle calculation (left/right rotation)
    const eyeDistance = Math.abs(leftEye.x - rightEye.x);
    const noseOffset = (noseTip.x - (leftEye.x + rightEye.x) / 2);
    const yaw = Math.atan2(noseOffset, eyeDistance) * (180 / Math.PI);

    return { yaw, pitch: 0, roll: 0 };
}
```

## ⚙️ 비디오 분할 & MariaDB 저장 시스템 구현

KYC 프로세스의 핵심인 **비디오 데이터 저장** 로직을 구현했습니다. 처음에는 서버에 전체 영상을 저장한 뒤 분할하려고 했지만, 단계별로 클라이언트에서 미리 쪼개서 보내는 편이 계산 효율이 좋지 않을까 고민했습니다. 결과적으로는 서버에서 분할하는 접근을 채택했습니다.

![System Architecture](/images/07-system-architecture-diagram.png)

시스템 아키텍처는 프론트엔드 React 컴포넌트 → 백엔드 Node.js API → MediaPipe 기반 컴퓨터 비전 처리로 이어지는 전체 데이터 흐름을 보여줍니다.

서버에서는 **fluent-ffmpeg**로 클라이언트가 올린 KYC 전체 비디오를 단계별로 분할했습니다. 처음에는 FFmpeg 환경변수 설정 문제로 500 에러가 발생했지만, 올바른 경로를 PATH로 등록하면서 해결했습니다.

![FFmpeg Video Splitting Process](/images/07-ffmpeg-video-split.png)

*입력부터 DB 저장까지의 FFmpeg 비디오 분할 워크플로우*

```javascript
// Video splitting using fluent-ffmpeg
ffmpeg(videoPath)
  .setStartTime(startTime)
  .setDuration(duration)
  .output(outputPath)
  .on('end', () => console.log('Video segment created successfully.'))
  .on('error', (err) => console.error('Error processing video:', err))
  .run();
```

### FFmpeg 환경 설정(Windows) 해결

Windows 환경에서 FFmpeg 경로 설정 문제를 해결하기 위해 아래 로직을 적용했습니다.

```javascript
// Automatic FFmpeg path setting in server.js
if (process.platform === 'win32' && !process.env.FFMPEG_PATH) {
    const username = os.userInfo().username;
    const ffmpegPath = `C:\\Users\\${username}\\AppData\\Local\\Microsoft\\WinGet\\Packages\\Gyan.FFmpeg_Microsoft.Winget.Source_8wekyb3d8bbwe\\ffmpeg-7.1.1-full_build\\bin\\ffmpeg.exe`;

    try {
        require('fs').accessSync(ffmpegPath);
        process.env.FFMPEG_PATH = ffmpegPath;
        console.log('🎬 FFmpeg path set:', ffmpegPath);
    } catch (error) {
        console.warn('⚠️ FFmpeg path not found. Searching in system PATH.');
    }
}
```

KYC 데이터를 체계적으로 관리하기 위해 **MariaDB**도 연동했습니다. 비디오 파일 자체를 BLOB로 DB에 넣는 대신, 파일 경로만 저장해 효율을 높였습니다.

![Database Schema](/images/07-database-schema-diagram.png)

DB 스키마는 사용자 정보, KYC 세션, 비디오 파일, 각도 측정, 감사 로그 테이블 간 관계를 보여줍니다. 각 테이블은 적절한 FK로 연결되어 데이터 정합성을 보장합니다.

```sql
CREATE TABLE kyc_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    session_id VARCHAR(100) NOT NULL UNIQUE,

    -- Whole video (store only file path)
    face_whole_path VARCHAR(500),
    face_whole_size INT,
    face_whole_duration DECIMAL(10,3),

    -- Step-by-step video (store only file path)
    face_left_path VARCHAR(500),
    face_right_path VARCHAR(500),
    face_close_path VARCHAR(500),

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

이렇게 하면 DB 용량이 불필요하게 커지지 않고, 향후 비디오 스트리밍 같은 기능 확장도 쉬워집니다.

### 비디오 녹화 시스템 개선

다양한 브라우저 환경에서도 안정적으로 동작하도록, 호환성 폴백을 구현했습니다.

```javascript
// Browser compatibility handling in video-recorder.js
class VideoRecorder {
    constructor() {
        // Fallback options for browser compatibility
        this.fallbackOptions = [
            'video/webm;codecs=vp9',
            'video/webm;codecs=vp8',
            'video/webm',
            'video/mp4'
        ];
    }

    getSupportedMimeType() {
        for (const mimeType of this.fallbackOptions) {
            if (MediaRecorder.isTypeSupported(mimeType)) {
                console.log(`✅ Supported MIME type: ${mimeType}`);
                return mimeType;
            }
        }
        console.warn('⚠️ No supported video MIME type found. Using default');
        return 'video/webm';
    }
}
```
