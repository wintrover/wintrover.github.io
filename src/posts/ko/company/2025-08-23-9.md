## Introduction

이 글에서는 레거시 애플리케이션을 React로 마이그레이션한 경험, 호환성을 높이기 위해 카메라 라이브러리를 교체한 과정, 그리고 신분증 인식을 위한 엣지 디텍션 알고리즘을 개선한 이야기를 공유합니다. 큰 기술적 변경과 전략적 의사결정이 함께 있었고, 그 결과 개발 프로세스와 제품 품질을 모두 끌어올릴 수 있었습니다.

### 1. 문제 — *왜 React 마이그레이션인가?*

레거시 애플리케이션은 바닐라 JavaScript와 jQuery로 만들어져 오랫동안 역할을 해왔지만, 다음과 같은 핵심 영역에서 한계를 드러내기 시작했습니다.

1. **코드 유지보수성** - 결합도가 높고 관심사 분리가 불명확해 유지보수가 점점 어려워짐
2. **사용자 경험** - UI가 구식으로 느껴지고, 현대 웹 애플리케이션에서 기대하는 부드러운 인터랙션이 부족
3. **개발 속도** - 신규 기능 추가가 점점 느려지고 실수가 늘어남
4. **테스트 난이도** - 컴포넌트 기반 구조가 없어 유닛 테스트가 어렵고 신뢰도가 낮음

### 2. 결정 — *왜 React인가?*

여러 옵션을 검토한 끝에 React를 선택했습니다. 이유는 꽤 명확했습니다.

#### 컴포넌트 기반 아키텍처
React의 컴포넌트 기반 접근은 모놀리식 애플리케이션을 재사용 가능한 단위로 분해할 수 있게 해줍니다. 결과적으로 코드가 더 유지보수하기 쉬워지고 이해하기도 쉬워집니다.

#### 생태계와 커뮤니티
React는 웹 개발 생태계에서 가장 큰 축 중 하나입니다. Redux/Zustand 같은 상태 관리부터 Material-UI/Ant Design 같은 UI 라이브러리까지 선택지가 풍부합니다.

#### 개발자 경험
JSX, hooks, virtual DOM 등으로 개발이 직관적이고 생산성이 높습니다.

#### 성능
virtual DOM과 효율적인 reconciliation 덕분에 UI 업데이트가 잦은 복잡한 화면에서도 성능이 좋습니다.

### 3. 마이그레이션 전략 — *어떻게 진행했나?*

마이그레이션 전략은 리스크를 최소화하고 비즈니스 연속성을 유지하는 방향으로 설계했습니다.

### 4. 기술적 난관 — *무엇을 배웠나?*

#### 상태 관리(State Management)
전환 과정에서 가장 어려웠던 것 중 하나가 상태 관리였습니다. 레거시 jQuery 컴포넌트와 React 컴포넌트가 공존해야 했기 때문에, 공용 이벤트 버스를 이용해 통신하는 하이브리드 방식을 적용했습니다.

```javascript
// Legacy jQuery to React communication
$(document).trigger('legacy-event', { data: someData });

// React component listening for legacy events
useEffect(() => {
  const handler = (event, data) => {
    // Handle legacy event
  };

  $(document).on('legacy-event', handler);

  return () => {
    $(document).off('legacy-event', handler);
  };
}, []);
```

#### 스타일 충돌(Styling Conflicts)
레거시 CSS와 React 컴포넌트 스타일이 충돌하는 문제도 컸습니다. 충돌을 막기 위해 엄격한 네이밍 컨벤션과 CSS Modules를 적용했습니다.

```javascript
// React component with CSS modules
import styles from './LegacyFeature.module.css';

function LegacyFeature() {
  return (
    <div className={styles.container}>
      {/* Component content */}
    </div>
  );
}
```

#### 성능 최적화(Performance Optimization)
기능을 더 많이 옮길수록 성능 이슈가 드러났고, 아래 최적화를 적용했습니다.

1. **코드 스플리팅** - 번들을 더 작은 청크로 분리
2. **레이지 로딩** - 필요한 시점에만 컴포넌트 로드
3. **메모이제이션** - React.memo/useMemo로 불필요한 리렌더 방지
4. **가상화(Virtualization)** - 대형 리스트에 windowing 적용

## 왜 react-camera-pro를 react-webcam으로 교체했나

### react-camera-pro의 문제

KYC 웹 애플리케이션을 개발하던 중, 브라우저 콘솔에서 `react-camera-pro` 관련 경고를 발견했습니다.

1. `it looks like an unknown prop "aspectRatio" is being sent through to the DOM`
2. `it looks like an unknown prop "mirrored" is being sent through to the DOM`
3. `Received 'false' for a non-boolean attribute 'mirrored'`

### 원인 분석

조사 결과 원인은 다음과 같았습니다.

- `react-camera-pro`는 `styled-components` v5 기반으로 작성됨
- 우리 프로젝트는 `styled-components` v6 사용
- v5와 v6 사이에 props 전달 방식 호환성 이슈가 존재
- v6에서는(v5와 달리) 기본적으로 props를 DOM 요소로 자동 전달하지 않음
- 그 결과 `react-camera-pro` 내부 컴포넌트가 DOM으로 잘못된 props를 넘겨 React 경고가 발생

### 카메라 라이브러리 비교

React 웹캠 라이브러리를 여러 개 검토했습니다.

1. **react-webcam** (채택)
   - 가장 대중적이고 유지보수가 활발한 React 웹캠 라이브러리
   - 기능이 풍부(비디오 녹화, 스냅샷, 비디오 피드 커스터마이징)
   - TypeScript 지원이 좋음
   - 커뮤니티가 크고 문서가 잘 되어 있음
   - 주간 다운로드: 약 28만(npm trends 기준)

2. **react-html5-camera-photo**
   - react-webcam 대안
   - HTML5 카메라 사진 촬영에 초점
   - 동영상 기능은 제한적
   - 주간 다운로드: 약 2.4만

3. **videojs-record**
   - Video.js 기반 비디오 녹화 플러그인
   - 다양한 미디어 포맷 지원
   - Video.js 의존으로 번들 크기 증가
   - 사진 캡처 기능은 제한적
   - 주간 다운로드: 약 2.1만

4. **webcamjs**
   - 가벼운 JavaScript 웹캠 라이브러리
   - React 통합 측면에서는 react-webcam보다 자연스럽지 않음
   - TypeScript 지원이 제한적일 수 있음
   - 주간 다운로드: 약 4.7천

### 왜 react-webcam이 가장 적합했나

프로젝트 요구사항은 다음과 같았습니다.
- 신분증 인증과 얼굴 인증을 위한 프론트엔드 KYC 웹 프로그램
- 신분증 이미지와 얼굴 녹화 영상을 백엔드로 전송 가능해야 함
- 스타트업 특성상 높은 개발 생산성 필요
- 참고 자료가 풍부해야 함
- 보안이 중요함

### 요구사항 기준 평가

1. **react-webcam**
   - **KYC 기능성**: 이미지 캡처 + 비디오 녹화 모두 지원
   - **백엔드 전송**: `getScreenshot()`로 이미지, 녹화된 비디오는 `blob` 이벤트로 획득 가능
   - **개발 생산성**: 가장 대중적이고 문서/예제가 풍부
   - **레퍼런스**: GitHub star 1,700+ / 주간 다운로드 28만+ / Stack Overflow 질문 다수
   - **보안**: 권한 관리가 좋고 HTTPS 환경으로 제한 가능

2. **react-html5-camera-photo**
   - **KYC 기능성**: 사진 촬영 특화, 비디오 기능은 제한적
   - **백엔드 전송**: 이미지 데이터는 가능하나 비디오 전송은 부족할 수 있음
   - **개발 생산성**: 문서가 상대적으로 적음
   - **레퍼런스**: GitHub star 200+ / 주간 다운로드 2.4만
   - **보안**: 기본적인 권한 관리 제공

3. **videojs-record**
   - **KYC 기능성**: 비디오 녹화에 특화, 이미지 캡처는 제한적
   - **백엔드 전송**: 비디오 데이터는 가능하나 이미지 캡처는 부족할 수 있음
   - **개발 생산성**: Video.js 학습 비용 발생
   - **레퍼런스**: GitHub star 1,400+ / 주간 다운로드 2.1만
   - **보안**: 권한 관리 제공

4. **webcamjs**
   - **KYC 기능성**: 이미지 캡처 + 비디오 녹화 제공
   - **백엔드 전송**: 이미지/비디오 데이터 획득 가능
   - **개발 생산성**: 문서가 상대적으로 적고 React 통합이 덜 자연스러움
   - **레퍼런스**: GitHub star 2,500+ / 주간 다운로드 4.7천
   - **보안**: 기본 권한 관리 제공

3. **CommonCamera를 react-webcam 기반으로 교체**
   - `src/components/CommonCamera.tsx` 수정

4. **CameraView 업데이트**
   - `src/components/capture/CameraView.tsx` 수정

5. **FaceCapturePage / IdCardCapturePage 업데이트**
   - `src/pages/FaceCapturePage.tsx`, `src/pages/IdCardCapturePage.tsx` 수정

6. **react-camera-pro 관련 코드 제거**

7. **테스트/검증**
   - 실행 후 기능 점검
   - 콘솔 경고 메시지 확인

이 계획대로 `react-camera-pro`를 `react-webcam`으로 교체하면 `styled-components` v6 호환성 문제를 해결할 수 있고, 더 널리 쓰이는 라이브러리를 기반으로 개발 생산성과 유지보수성을 함께 개선할 수 있습니다.

## CannyEdgeCanvas.tsx 개선 과정

### 초기 문제

CannyEdgeCanvas.tsx 컴포넌트는 웹캠 비디오 입력에 Canny 엣지 디텍션을 적용해 신분증 외곽선을 검출하고 사각형을 그립니다. 초기 구현에는 다음 문제들이 있었습니다.

1. **히스토그램 평활화 과적용**: 이미 대비가 큰 상황(흰색 신분증 + 검정 배경)에서 평활화를 하면 오히려 엣지 검출이 어려워짐
2. **가우시안 블러가 과함**: 21x21 커널은 엣지를 뭉개 검출이 어려워짐
3. **Canny 임계값 부적절**: 고정 임계값(50/150)이 조명 조건에 따라 최적이 아닐 수 있음
4. **엣지 연결성 부족**: 점선처럼 끊겨 보여 사각형을 안정적으로 그리기 어려움

### 개선 과정

#### 1차 개선: 히스토그램 평활화 제거 및 파라미터 조정

1. **히스토그램 평활화 제거**: 원본 그레이스케일 이미지를 그대로 사용
2. **가우시안 블러 감소**: 커널 21x21 → 5x5로 낮춰 엣지가 뭉개지지 않게 함
3. **Canny 임계값 조정**: low 50 → 100, high 150 → 200으로 높여 정확도를 개선

이 변경으로 흰 신분증 + 검정 배경 환경에서도 엣지 검출이 정상적으로 동작하도록 개선되었습니다.

#### 2차 개선: 형태학 연산 적용 및 파라미터 재조정

1. **형태학 연산 적용**: 엣지 연결성을 높이기 위해 `cv.MORPH_CLOSE` 적용
2. **커널 크기 증가**: 3x3 → 5x5로 늘려 연결성 강화
3. **Canny 임계값 재조정**: low 100 → 80, high 200 → 160으로 조정해 민감도를 높임

이 변경으로 엣지가 더 잘 연결되어 사각형을 훨씬 안정적으로 그릴 수 있게 되었습니다.

### 최종 결과

개선을 통해 CannyEdgeCanvas.tsx는 다음 효과를 얻었습니다.

- 흰 신분증 + 검정 배경에서도 안정적 엣지 검출
- 엣지 연결성 개선으로 사각형을 더 안정적으로 드로잉
- 다양한 조명 조건에서도 일관된 성능 유지

이제 CannyEdgeCanvas.tsx는 신분증 인식에 필요한 기능을 안정적으로 수행할 수 있습니다.
