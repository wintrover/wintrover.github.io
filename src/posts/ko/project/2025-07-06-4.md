## 소개

개발에서 코드 품질 관리는 선택이 아니라 필수입니다. 특히 **Svelte + Django 모노레포** 같은 구조에서는 정적 분석 도구가 없으면 품질 기준을 일관되게 유지하기가 어렵습니다. 이 글에서는 SonarQube Community Edition을 로컬에 구성하고, **버그 0개 + Grade A**까지 끌어올린 과정을 공유합니다.

> “코드 품질은 옵션이 아니라 의무다. SonarQube는 우리가 필요로 하는 객관적 지표를 제공한다.”

---

### 1. SonarQube Community Edition vs SonarCloud

| 항목 | Community Edition | SonarCloud |
|---------|-------------------|------------|
| 비용 | 무료 | 공개 레포는 무료, 비공개는 유료 |
| 설치 | 로컬 Docker 구성 | 클라우드 서비스 |
| PR Decoration | 불가(Developer+ 필요) | 가능 |
| 데이터 보안 | 로컬 저장 | 외부 서버 |
| 커스터마이즈 | 제한적 | 제한적 |

개인 프로젝트나 보안이 중요한 환경에서는 **Community Edition**이 특히 적합합니다.

### 2. Docker로 SonarQube 올리기

#### docker-compose.yml 설정

```yaml
version: "3.9"
services:
  db-sq:
    image: postgres:15
    environment:
      POSTGRES_USER: sonar
      POSTGRES_PASSWORD: sonar
      POSTGRES_DB: sonar
    volumes:
      - postgresql:/var/lib/postgresql
      - postgresql_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U sonar"]
      interval: 30s
      timeout: 10s
      retries: 3

  sonarqube:
    image: sonarqube:community
    depends_on:
      db-sq:
        condition: service_healthy
    environment:
      SONAR_JDBC_URL: jdbc:postgresql://db-sq:5432/sonar
      SONAR_JDBC_USERNAME: sonar
      SONAR_JDBC_PASSWORD: sonar
    volumes:
      - sonarqube_data:/opt/sonarqube/data
      - sonarqube_extensions:/opt/sonarqube/extensions
      - sonarqube_logs:/opt/sonarqube/logs
    ports:
      - "9000:9000"

volumes:
  sonarqube_data:
  sonarqube_extensions:
  sonarqube_logs:
  postgresql:
  postgresql_data:
```

#### 서버 초기화

```bash
docker compose up -d
```

초기 기동에는 2~3분 정도 걸릴 수 있고, `http://localhost:9000`에서 접근합니다.

### 3. 프로젝트 설정 & 분석 실행

#### sonar-project.properties 만들기

```properties
sonar.projectKey=frontend
sonar.projectName=Frontend
sonar.sources=client,core
sonar.exclusions=**/node_modules/**,**/__pycache__/**,**/migrations/**
sonar.python.version=3.11
sonar.host.url=http://localhost:9000
```

#### SonarScanner CLI 설치 & 실행

```bash
# Download SonarScanner for Windows
curl -o sonar-scanner.zip https://binaries.sonarsource.com/Distribution/sonar-scanner-cli/sonar-scanner-cli-6.2.1.4610-windows-x64.zip
tar -xf sonar-scanner.zip

# Run analysis
.\sonar-scanner-6.2.1.4610-windows-x64\bin\sonar-scanner.bat -D"sonar.login=<token>"
```

### 4. 발견된 이슈 & 해결 과정

#### 최초 분석 결과

| 지표 | 값 | 등급 |
|--------|-------|-------|
| Bugs | 1 | C |
| Code Smells | 3 | A |
| Vulnerabilities | 0 | A |
| Duplications | 0% | A |
| Test Coverage | 0% | - |

#### 핵심 이슈

1. **HTML title 태그 누락** (Bug)
   ```html
   <!-- Before -->
   <head>
   	<meta charset="utf-8" />
   	<link rel="icon" href="%sveltekit.assets%/favicon.png" />
   	<meta name="viewport" content="width=device-width, initial-scale=1" />
   	%sveltekit.head%
   </head>

   <!-- After -->
   <head>
   	<meta charset="utf-8" />
   	<title>CV Factory - Professional Resume Builder</title>
   	<meta name="description" content="Create professional resumes with our easy-to-use CV builder" />
   	<link rel="icon" href="%sveltekit.assets%/favicon.png" />
   	<meta name="viewport" content="width=device-width, initial-scale=1" />
   	%sveltekit.head%
   </head>
   ```

2. **사용하지 않는 TypeScript import** (Code Smell)
   ```typescript
   // Before
   import { page } from '$app/stores';
   import type { LayoutLoad } from './$types';

   // After
   import type { LayoutLoad } from './$types';
   ```

3. **Logical OR 대신 Nullish Coalescing 권장** (Code Smell)
   ```typescript
   // Before
   export const prerender = data.prerender || false;

   // After
   export const prerender = data.prerender ?? false;
   ```

4. **사용하지 않는 Python 파라미터** (Code Smell)
   ```python
   # Before
   def home(request):
       return render(request, 'core/home.html')

   # After
   def home(_request):
       return render(_request, 'core/home.html')
   ```

### 6. GitHub Actions vs 로컬 분석

#### GitHub Actions의 한계

```yaml
# Example of failing workflow
env:
  SONAR_HOST_URL: "http://localhost:9000"
  SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
```

**문제점:**
- GitHub hosted runner에서 `localhost:9000` 접근 불가
- `Connection refused`가 쉽게 발생

#### 대안

1. **ngrok 터널링** — 보안 위험, 불안정
2. **포트 포워딩** — 네트워크 설정이 복잡
3. **로컬에서만 실행** — 가장 안전하고 단순 ✅

```bash
# Local analysis command
sonar-scanner \
  -Dsonar.projectKey=frontend \
  -Dsonar.sources=client,core \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.login=<token>
```

## 결론

SonarQube Community Edition만으로도 **버그 0개 + Grade A 품질**을 달성할 수 있었습니다. PR Decoration 같은 고급 기능은 제한되지만, 로컬 환경에서의 품질 관리 체계는 충분히 구축 가능합니다.

핵심 성과:
- 🐛 **버그 100% 해결** (1 → 0)
- 🧹 **코드 스멜 완전 제거** (3 → 0)
- 📈 **신뢰도 등급 2단계 상승** (C → A)
- 🗂️ **프로젝트 구조 최적화** (55MB+ 절감)

> 💡 **배운 점**: 도구는 수단일 뿐입니다. 중요한 건 객관적인 지표를 기반으로 꾸준히 개선하고, 팀이 같은 목표로 움직이는 것입니다.

코드 품질은 하루아침에 만들어지지 않습니다. SonarQube 같은 도구로 “측정 가능하게” 만들고, 개선을 습관으로 만드는 것이 핵심입니다.
